# UI List — React + TypeScript + Vite

Небольшое SPA для просмотра списка элементов с поиском и сортировкой. Данные эмулируются локальным API и подгружаются с задержкой для имитации запроса.

## Возможности
- Поиск по названию в реальном времени.
- Сортировка по алфавиту (A–Z / Z–A) с учетом локали `ru`.
- Мгновенный отклик (фильтрация и сортировка выполняются на клиенте).
- Демонстрационная задержка загрузки данных (1 секунда).

## Технологии
- React 19 + TypeScript
- Vite 7
- ESLint 9

## Требования
- Node.js 18+ (рекомендуется LTS)
- npm (из поставки Node.js)

## Установка и запуск
```bash
npm install
npm run dev
```
После запуска откройте адрес из терминала (по умолчанию `http://localhost:5173`).

Полезные скрипты:
- `npm run dev` — режим разработки с HMR.
- `npm run build` — сборка проекта.
- `npm run preview` — предпросмотр собранной версии.
- `npm run lint` — проверка кода ESLint.

## Структура проекта
```
src/
├─ App.tsx        # UI: поиск и сортировка списка
├─ App.css        # Стили компонента
├─ api.ts         # Эмуляция API с задержкой
├─ fetch.ts       # Функция загрузки данных
├─ data.ts        # Статичный набор данных
├─ main.tsx       # Точка входа приложения
└─ index.css      # Глобальные стили
```

## Как это работает
- `api.ts` возвращает промис с данными через 1 секунду.
- `fetch.ts` инкапсулирует вызов API и обработку ошибок.
- `App.tsx` подгружает данные, хранит выбранное направление сортировки (`ASC`, `DESK`, `NOT`) и строку поиска, а затем:
  - фильтрует список по `title` с учетом регистра,
  - сортирует с использованием `localeCompare` для русской локали,
  - отображает результат.

## Идеи для улучшений
- Добавить сортировку по времени или тексту.
